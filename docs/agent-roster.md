## Current state rule

A named role is not considered active merely because this file exists.
A role becomes active only when:

1. its contract exists
2. Milo delegates responsibility explicitly
3. a live session or concrete task handoff is created

## Operating rule

Milo must distinguish clearly between:
- defined roles
